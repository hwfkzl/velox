# Velox 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
Velox

### 1.2 一句话描述
基于 Flutter 开发的跨平台 VPN 客户端应用,对接 V2Board 后端管理系统,为用户提供安全、稳定、易用的网络代理服务。

### 1.3 目标用户
- 需要网络隐私保护的个人用户
- 需要访问特定区域内容的用户
- 对网络安全有较高要求的商务人士
- 技术爱好者和开发者

### 1.4 核心价值
- **安全性**: 支持多种加密协议,保护用户隐私
- **稳定性**: 智能节点选择,确保连接质量
- **便捷性**: 一键连接,操作简单
- **跨平台**: 统一体验覆盖 iOS、Android、macOS、Windows

---

## 2. 平台支持

### 2.1 移动端
- **iOS**: 支持 iOS 13.0 及以上版本
- **Android**: 支持 Android 6.0 (API 23) 及以上版本

### 2.2 桌面端
- **macOS**: 支持 macOS 10.15 及以上版本
- **Windows**: 支持 Windows 10 及以上版本

### 2.3 技术栈
- **开发框架**: Flutter 3.x
- **开发语言**: Dart
- **后端对接**: V2Board (基于 wyx2685 的 v2board webman 版本)

## 3. 多语言支持

### 3.1 支持语言
1. **俄语 (Русский)** - 主要市场
2. **英语 (English)** - 国际通用
3. **繁体中文 (繁體中文)** - 港澳台市场

## 4. 后端集成

### 4.1 后端系统
- **系统**: V2Board (xiaoV2b 版本)
- **仓库地址**: https://github.com/vlesstop/xiaoV2b
- **基础架构**: 基于 wyx2685 的 v2board,支持 webman

### 4.2 API 对接要求

#### 4.2.1 用户认证模块
```
POST /api/v1/passport/auth/register
POST /api/v1/passport/auth/login
POST /api/v1/passport/auth/logout
GET  /api/v1/user/info
POST /api/v1/user/update
```

**功能要求**:
- 邮箱注册/登录
- 邀请码注册
- 验证码验证
- Token 认证机制
- 自动登录(记住密码)
- 登出清理本地缓存

#### 4.2.2 订阅管理模块
```
GET  /api/v1/user/getSubscribe
GET  /api/v1/user/plan/fetch
POST /api/v1/user/order/save
GET  /api/v1/user/order/details
```

**功能要求**:
- 获取用户订阅信息
- 查看可用套餐列表
- 购买/续费套餐
- 订单历史查询
- 流量使用统计
- 订阅到期提醒

#### 4.2.3 节点服务模块
```
GET  /api/v1/client/subscribe?token={token}
```

**功能要求**:
- 获取节点列表
- 解析订阅链接
- 支持协议:
  - Shadowsocks
  - VMess
  - VLESS
  - Trojan
  - Hysteria
  - Hysteria2

#### 4.2.4 流量报告
```
POST /api/v1/user/server/reportTraffic
```

**功能要求**:
- 实时上报流量使用
- 流量统计准确性
- 断线重连后继续统计

#### 4.2.5 通知公告
```
GET  /api/v1/user/notice/fetch
```

**功能要求**:
- 获取系统公告
- 消息推送
- 维护通知

## 7. 性能要求

### 7.1 应用性能
- **启动时间**: 冷启动 < 3s,热启动 < 1s
- **连接速度**: VPN 连接建立 < 5s
- **内存占用**:
  - 移动端: < 150MB
  - 桌面端: < 200MB
- **CPU 占用**: 空闲时 < 5%,连接时 < 15%
- **电池消耗**: 后台运行 24h < 10%(移动端)

### 7.2 网络性能
- **延迟增加**: < 50ms(相对直连)
- **速度损失**: < 10%(相对直连)
- **稳定性**: 连续运行 24h 不掉线
- **重连时间**: 断线重连 < 10s

### 7.3 界面性能
- **界面流畅度**: 60 FPS
- **列表滚动**: 无卡顿
- **动画流畅**: 过渡动画 < 300ms

## 15. 风险与挑战

### 15.1 技术风险
- **VPN 平台差异**: 各平台 VPN 实现机制不同,需要大量原生代码
- **协议兼容性**: 多种代理协议的完整支持和测试
- **性能优化**: 保证 VPN 连接稳定性和速度
- **应用商店审核**: iOS App Store 对 VPN 应用审核严格

**应对措施**:
- 提前调研各平台 VPN 实现方案
- 建立完善的测试环境
- 遵守平台政策,准备合规材料

### 15.2 市场风险
- **竞品压力**: VPN 市场竞争激烈
- **政策变化**: 各地区对 VPN 的政策可能变化
- **用户获取**: 冷启动阶段用户获取成本高

**应对措施**:
- 差异化定位,提供独特价值
- 关注政策动态,及时调整策略
- 制定用户增长计划

### 15.3 运营风险
- **服务器成本**: 节点服务器维护成本
- **客服压力**: 用户问题支持
- **滥用风险**: 服务被滥用导致封禁

**应对措施**:
- 合理定价,覆盖成本
- 建立完善的帮助文档和自助系统
- 实施使用限制和监控机制
