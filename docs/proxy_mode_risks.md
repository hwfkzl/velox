# 代理模式风险分析报告

> 更新时间：2026-04-01  
> 适用版本：Velox（velox）当前版本

---

## 已修复的问题

| 问题 | 修复方式 |
|------|---------|
| 全局模式局域网不可访问 | `global` 分支加 `ipIsPrivate: true → direct` |
| 直连模式 UI 误导用户 | 从用户界面隐藏直连选项，老用户数据迁移到规则模式 |
| 首次启动规则集下载失败 | 打包 `.srs` 文件为 App Assets，3天后台CDN更新 |
| GeoSite/GeoIP 更新间隔过长 | 从7天缩短至3天 |

---

## 剩余风险

### 🔴 高危

#### 1. 桌面端 WebRTC IP 泄露
- **影响平台**：macOS / Windows / Linux
- **原因**：桌面端使用系统代理模式，浏览器 WebRTC 协议绕过系统代理，直接使用真实网卡通信，导致真实 IP 暴露给网页
- **影响**：用户以为 IP 已被代理隐藏，实际 WebRTC 探测到真实 IP
- **iOS / Android**：不受影响（TUN 模式接管全部流量）
- **修复难度**：🔴 高——需要在原生层实现 TUN 模式（macOS 需 SMJobBless 特权 Helper，Windows 需捆绑 Wintun 驱动）
- **当前处理**：UI 提示用户桌面端系统代理的已知局限
- **行业现状**：ClashX、Surge、Clash for Windows 均以"增强模式/TUN模式"作为可选功能，默认模式同样存在此问题

---

#### 2. 桌面端：不遵守系统代理的 App
- **影响平台**：macOS / Windows / Linux
- **原因**：系统代理只对主动读取系统代理设置的 App 生效，游戏客户端、部分 Electron App、命令行工具（curl、git 等）默认不走系统代理
- **影响**：这类 App 的流量完全绕过代理，真实 IP 暴露
- **iOS / Android**：不受影响（TUN 模式）
- **修复难度**：🔴 高——同上，需原生层 TUN 模式
- **当前处理**：UI 提示

---

### 🟡 中危

#### 3. 全局模式：国内银行/支付 App 报风险
- **影响平台**：全平台
- **原因**：全局模式下所有流量出口为境外 IP，国内银行 App、微信支付通过 IP 检测到境外地址，触发安全拦截
- **影响**：支付失败、弹出异地登录警告、要求人脸/短信验证
- **修复难度**：🟢 低——加 UI 说明文字即可
- **状态**：待加 UI 提示

---

#### 4. 自动切换死节点死循环
- **影响平台**：全平台
- **原因**：死节点检测（8秒）+ 自动切换逻辑中，若所有节点均不可用，会在死节点之间无限循环切换
- **影响**：每次循环消耗约 10 秒，用户体验差，无法自动恢复
- **修复难度**：🟢 低——加最大重试次数限制（建议3次），超出后停止并提示用户
- **状态**：待修复

---

### 🟢 低危

#### 5. IPv6 覆盖不完整
- **影响平台**：全平台
- **原因**：GeoIP-CN 对 IPv6 地址段收录不如 IPv4 完整
- **影响**：部分中国 IPv6 地址被误判走代理（速度变慢），部分境外 IPv6 地址被误判直连（隐私影响极小）
- **修复难度**：🔴 高——依赖上游数据库完善，App 层面无法解决
- **状态**：接受，依赖 SagerNet 上游更新

---

#### 6. 代理端口冲突无提示
- **影响平台**：桌面端
- **原因**：sing-box 占用 `10808`（代理）和 `19090`（Clash API）两个端口，若本机已有其他程序占用，启动失败但错误信息模糊
- **影响**：用户看到"连接失败"，不知道原因
- **修复难度**：🟢 低——捕获端口冲突错误，显示具体提示
- **状态**：待修复

---

#### 7. 规则集文件损坏无降级处理
- **影响平台**：macOS / Windows / Linux / Android
- **原因**：本地缓存的 `.srs` 文件损坏时，sing-box 启动失败，没有自动回退到 remote 模式的机制
- **影响**：极少见，通常文件损坏概率极低
- **修复难度**：🟡 中——在 `RuleSetManager` 中加文件完整性校验，损坏时删除并回退到 remote
- **状态**：待修复

---

## 修复优先级建议

| 优先级 | 问题 | 改动大小 |
|--------|------|---------|
| P1 | 自动切换死循环（加最大重试次数） | 小 |
| P1 | 全局模式 UI 加银行/支付提示 | 小 |
| P2 | 端口冲突加具体错误提示 | 小 |
| P2 | 规则集损坏降级处理 | 中 |
| P3 | WebRTC 泄露 + 代理绕过（桌面 TUN 模式） | 极大，规划为未来版本 |

---

## 桌面端已知局限说明（建议 UI 展示）

> ⚠️ 桌面端当前使用系统代理模式。浏览器 WebRTC 流量及不支持系统代理的应用（游戏、部分命令行工具）不经过代理，为已知限制。完整流量接管（TUN 模式）将在未来版本支持。
